BDCHM Interactive Documentation

Design explorations for the BioData Catalyst Harmonized Model schema explorer

We're redesigning the interactive schema documentation based on stakeholder feedback. The current view shows too much at once; the new direction uses progressive disclosure — start simple, drill into detail on demand. Below are mockups exploring different approaches. We'd love your feedback on what's useful and what isn't.

Progressive Disclosure (Tabular Drilldown)

The proposed new default view. Entities organized by category, with inline drilldowns for slots, variables, and range details. This is a static mockup showing a sequence of narrated examples — not yet interactive beyond category expand/collapse and dark mode.

Tabular Drilldown Mockup
Seven narrated examples walking through the proposed UI: default view with pinned entities, expanding categories, slot drilldown with inherited/overridden tags, clicking enum and class range badges for inline detail cards, and variable drilldowns (rich vs. simple). Includes UI controls mockup (search, column toggles, expand/collapse) and a design notes section.
progressive disclosure light + dark mode 7 examples primary direction

Model Structure: Containment Hierarchy

BDCHM uses relational FK-style references (child points to parent), so we invert certain edges to show "what contains what." These views are proof-of-concept explorations, not proposed final designs. The containment hierarchy can inform navigation, default groupings, and the "pin entities to generate a focused diagram" feature.

Containment Tree (Indented List)
Interactive indented list rooted at ResearchStudyCollection showing the full containment hierarchy. FK-direction edges are inverted for the containment view. Inheritance (is-a) subclasses are nested inline with a diamond marker and only show their own additional slots — no inherited duplication. Expand/collapse, hide subclasses toggle, dark mode.
containment is-a + has-a FK inversion proof of concept
Containment Graph (Node-Link Diagram)
Cytoscape + dagre graph visualization of the same containment data (49 nodes, 95 edges). Three visually distinct edge types: solid (has-a), purple (flipped FK), and dashed green (is-a subclass). Click a node to highlight its neighborhood and see details in a side panel. Toggle has-a and is-a edges independently. Multiple layout options (top-down, left-right, force-directed).

Current App

Current Interactive Schema Explorer
The existing app with two/three-panel layout, link overlays connecting classes to their ranges, detail popups, and variable tables. This is the "kitchen sink" view that stakeholders found overwhelming — we're exploring alternatives above. Still useful for detailed exploration and as a reference for what information needs to be accessible in the redesign.

We'd like your feedback! Some questions to consider as you explore:
  • Does the progressive disclosure approach (tabular drilldown) feel like the right direction?
  • Is the containment tree useful for understanding model structure? Does the hierarchy make sense, or are some edges pointing the wrong way?
  • Should the "current app" link-based view be kept as an option (e.g., a "Kitchen Sink" tab), or does the new design fully replace it?
  • Would you use a "pin entities of interest, generate a focused diagram" feature?
  • What's confusing, missing, or that you wish worked differently?
